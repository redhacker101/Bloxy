Getting Started

Bloxy games are built using Godot 4.6.2. Any other versions will be rejected. Developers create the game world, scripts, UI, assets, and gameplay inside Godot, then export the project into the format required by Bloxy.

Bloxy Runtime

Bloxy games do not run as completely random standalone files. They run through Godot Engine. 

Bloxy App
  ↓
Bloxy Runtime(Godot Engine)
  ↓
client.pck
  ↓
server.pck

Bloxy SDK

Gives the game access to player data, username, user ID, avatar mode, and server session.

Avatar Loader

Loads Bloxy avatars or lets the game choose custom character handling. This will be handled by the Bloxy SDK

Network Manager

Handles join tokens, connection state, player joined, and player left events. This is apart of the Bloxy SDK

Server Bridge

Lets the game communicate safely with Bloxy backend systems.

Avatar System

Bloxy provides a default avatar system. Games can use the player’s normal Bloxy avatar, use a custom avatar system, or use no avatar at all depending on the game type.

Bloxy Avatar

The game uses the player’s normal Bloxy avatar loaded by the app.

Custom Avatar

The game uses its own character system, like cars, soldiers, monsters, animals, or custom rigs.

No Avatar

The game does not spawn a player character. Useful for menus, strategy games, or UI-based games.

Avatar Modes:
- bloxy_avatar
- custom_avatar
- no_avatar

Multiplayer / Join & Leave

Join and leave events are handled by the Bloxy Runtime and Bloxy server systems. Games should not fake player identity or manually trust usernames from the client. 

User clicks Play
  ↓
Bloxy creates launch token
  ↓
Runtime launches client.pck
  ↓
Game connects to server
  ↓
Server validates token
  ↓
Player joins game

The server validates the user, checks if they are banned, checks if the game is valid, and then tells the game when a player joins or leaves.

Required Upload Format

Bloxy games must be uploaded in a zip file with the correct structure. 

 
Multiplayer:

  my_bloxy_game.zip
├── manifest.json
├── client/
│   └── client.pck
└── server/
    └── server.pck

{
  "title": "Test Game",
  "description": "A basic Bloxy test game",
  "creator": "ali",
  "godot_version": "4.6.2",
  "max_players": 20,
  "multiplayer": true
}

Singleplayer:

manifest.json
client/
  client.pck

{
