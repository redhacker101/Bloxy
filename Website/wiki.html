Getting Started

Bloxy games are built using Godot. Developers create the game world, scripts, UI, assets, and gameplay inside Godot, then export the project into the format required by Bloxy.

During Alpha, Bloxy only accepts multiplayer games that can run on Linux servers. Each game must include a client pack for players and a server pack for Bloxy servers.

Bloxy Runtime

Bloxy games do not run as completely random standalone files. They run through the Bloxy Runtime. The runtime is the layer between the Bloxy app, the Godot game package, and the Bloxy servers. 

Bloxy App
  ↓
Bloxy Runtime
  ↓
client.pck
  ↓
Bloxy Game Server / APIs

The runtime handles platform systems such as player identity, launch tokens, join/leave events, avatar loading, and communication with Bloxy servers.

Bloxy Plugins

Developers will use Bloxy plugins or SDK files inside their Godot project. These plugins allow the game to communicate with Bloxy systems without developers having to build login, avatars, or platform networking from scratch.

Bloxy SDK

Gives the game access to player data, username, user ID, avatar mode, and server session.

Avatar Loader

Loads Bloxy avatars or lets the game choose custom character handling.

Network Manager

Handles join tokens, connection state, player joined, and player left events.

Server Bridge

Lets the game communicate safely with Bloxy backend systems.

Avatar System

Bloxy provides a default avatar system. Games can use the player’s normal Bloxy avatar, use a custom avatar system, or use no avatar at all depending on the game type.

Bloxy Avatar

The game uses the player’s normal Bloxy avatar loaded by the runtime.

Custom Avatar

The game uses its own character system, like cars, soldiers, monsters, animals, or custom rigs.

No Avatar

The game does not spawn a player character. Useful for menus, strategy games, or UI-based games.

Avatar Modes:
- bloxy_avatar
- custom_avatar
- no_avatar

Multiplayer / Join & Leave

Join and leave events are handled by the Bloxy Runtime and Bloxy server systems. Games should not fake player identity or manually trust usernames from the client. 

User clicks Play
  ↓
Bloxy creates launch token
  ↓
Runtime launches client.pck
  ↓
Game connects to server
  ↓
Server validates token
  ↓
Player joins game

The server validates the user, checks if they are banned, checks if the game is valid, and then tells the game when a player joins or leaves.

Required Upload Format

Bloxy games must be uploaded in a zip file with the correct structure. 

my_bloxy_game.zip
├── client.pck
├── server.pck
├── manifest.json
├── icon.png
└── screenshots/
    ├── 1.png
    └── 2.png
